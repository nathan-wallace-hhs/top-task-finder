# Proposed GitHub Issues

This directory contains three proposed GitHub Issues inspired by the techniques described in the [GitHub Accessibility — Getting Started with Agents](https://accessibility.github.com/documentation/guide/getting-started-with-agents/) guide. Each file is formatted as a ready-to-open GitHub Issue.

## Issues

| File | Title | Labels |
|------|-------|--------|
| [`issue-copilot-instructions.md`](issue-copilot-instructions.md) | Add `.github/copilot-instructions.md` to prepare the codebase for Copilot coding agent use | `enhancement`, `ai-agent`, `documentation` |
| [`issue-ai-semantic-url-scoring.md`](issue-ai-semantic-url-scoring.md) | Use on-device AI agent to semantically score discovered URLs by user-task relevance | `enhancement`, `ai-agent`, `url-discovery` |
| [`issue-wcag-em-agent-workflow.md`](issue-wcag-em-agent-workflow.md) | Add a GitHub Actions workflow for agent-powered WCAG-EM evaluation project setup | `enhancement`, `ai-agent`, `wcag-em`, `github-actions` |

## How to create these as GitHub Issues

1. Open each markdown file.
2. Copy the body (everything after the first heading and labels line).
3. Navigate to **Issues → New Issue** in this repository.
4. Paste the title (the first `#` heading) and body.
5. Apply the labels listed in the **Labels** field at the top of each file.
